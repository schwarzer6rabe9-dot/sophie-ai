## Architecture

**Single-server deployment**: Flask (`server.py`) serves both the REST API and the compiled React frontend as static files. In production this runs on Render (`https://sophie-ai-jfna.onrender.com`).

**Frontend**: The `API` constant in `frontend/src/App.jsx:4` is hardcoded to the production Render URL. For local dev, either run against production or change this constant to `http://localhost:5001`.

**Memory system**: Two-tier persistence â€” JSONBin cloud API (primary) with `memory.json` local file as fallback. Memory holds three collections:
- `facts`: permanent facts about Antonio (capped at 50)
- `recent`: last 10 conversation snippets (cleared daily by auto-summary)
- `summaries`: 30-day rolling daily summaries generated by Claude at 22:00 Zurich time

The background thread in `server.py:38` handles auto-summary at 22:00 and auto-save every 4 hours.

**Chat flow** (`App.jsx:156`): The frontend classifies messages into three paths before hitting `/chat`:
1. n8n keywords â†’ `/n8n` endpoint â†’ forwards to n8n cloud webhook
2. Email/mail keywords â†’ fetches `/gmail/unread` first, then passes result to `/chat`
3. All other messages â†’ `/chat` directly

**Google OAuth**: Token exchange happens server-side at `/oauth/callback`. The resulting token (access + refresh) is stored inside `memory.json` under `google_token`, not in a separate secrets store.

**TTS**: ElevenLabs `eleven_multilingual_v2` model with voice ID `7eVMgwCnXydb3CikjV7a` (Lea). Audio is written to `/tmp/speech.mp3` and streamed back.

## Model usage

All Claude calls use `claude-sonnet-4-6` with `max_tokens=300`. The system prompt injects current Zurich time, stored facts, last 7 daily summaries, and the 5 most recent conversation snippets.
